RLCC Assistant Guide

Documentation

Complete guide for administrators and superusers to operate RLCC Assistant.

Admin Docs
Quick Start
  1. Open Dashboard to review system counts and bug metrics.
  2. Use Members to manage user roles.
  3. Configure Group Types before creating circles.
  4. Moderate discussions from Group Topics.
Access Model
  • SUPERUSER: Full system controls and integrations.
  • ADMINISTRATOR: Member, content, and operations management.
  • ACCOUNTANT: Giving and tithe visibility focused tasks.
  • ENDUSER: Personal profile, circles, and submissions only.

Main Navigation

Area Purpose Who Uses It
Dashboard System overview and operational metrics. All authenticated users
Giving Create, review, and maintain tithe records and types. Admin, Superuser, Accountant
Groups Manage circles, members, chats, and topic discussions. All users (scope by role)
Members Search members and assign roles per user. Admin and Superuser
Administrator Tools System settings, bug monitoring, integrations. Admin and Superuser

Member Management Workflow

  1. Open Members from the header.
  2. Search by name, email, ID, or PCOID.
  3. Select a member row and assign/remove roles.
  4. Save roles and confirm updated badges in the table.

Role updates can refresh permissions immediately after token refresh.

Group Topics & Comments

  • Only circle members, administrators, and superusers can comment.
  • Comments are sorted newest first with relative timestamps.
  • Replies are hidden by default and expandable in batches.
  • Delete is allowed only when ownership/rules are satisfied.

Operations & Maintenance

System Settings

Use Settings to toggle maintenance mode and dev mode.

Reported Bugs

Track incoming bug reports and monitor resolution progress.

Queues & Migration

Superusers can monitor AMQP queues and run PCO user migration.

Troubleshooting

  • Empty tables: verify role permissions and active filters.
  • Access denied: confirm user has required role and relogin after role changes.
  • Real-time chat issues: verify socket token validity and circle membership.
  • Slow data refresh: retry after cache TTL or re-open the page.
